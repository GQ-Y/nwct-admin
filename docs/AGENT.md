工作流程要求：
Go 项目完整开发工作流程
编写/修改代码
使用编辑器或 IDE 编写或更新 Go 源文件。
编译检查
执行 go build 或 go run .
若编译失败 → 跳转到第 3 步；若成功 → 进入第 4 步。
修复编译错误
根据编译器输出的错误信息（如语法错误、未定义变量、类型不匹配等）修正代码。
重复第 2 步，直至编译通过。
启动服务
运行可执行文件或直接 go run . 启动应用（如 HTTP 服务）。
确保服务在指定端口正常监听。
接口调用测试（使用 curl）
使用 curl 发送请求测试接口行为，例如：
bash
1
验证响应状态码、响应体、头信息是否符合预期。
分析测试结果
若接口行为正确 → 开发完成或进入下一轮功能开发。
若接口返回错误、崩溃或逻辑不符 → 跳转到第 7 步。
修复运行时/逻辑错误
查看服务日志或 panic 信息，定位问题（如空指针、路径错误、逻辑缺陷等）。
修改代码并返回第 2 步，重新编译验证。
重复迭代
持续循环“修改 → 编译 → 启动 → 测试 → 修复”过程，直至功能稳定。
